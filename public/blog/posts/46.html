Developing a Graphical Standings Tool

Part 3: More modularization

2025-12-14

Where we're at right now

Right now I've got an outline of how our graphic should look, as well as some foundations of how we're going to make it. I also have a wrapper class, Element (previously called SVGElement), on ET.Element which allows us to more easily create elements with content within them. Finally, I've figured out how to use an external CSS stylesheet within my generated SVG code, which should make styling a lot easier.

Defining more classes to help us out

Now, and this is part of what's taken so long, I have to make a decision for how to proceed. Should I define a class? I should probably define a few.

The first class I'm defining is gonna be a wrapper around my Element class defining specifically an SVG element. Each svg element I make should have pretty much all the same attributes, except for possibly the width and the height, so this should simplify things.

Then, a class to handle creation of our GSTool graphic. I'll call it GSToolGraphic, and I'll let it first essentially create skeleton.svg on itself. I'm also realizing that I'm also gonna have to generate a CSS file as well! Let's leave that for later though.

Importing from parent directory

Since I want to write testing scripts, but not have them in the root directory, I've had to figure out how to import a python file from a parent directory. Thankfully, GeeksforGeeks has a tutorial which was easy to follow and understand.

Result of today's work

Hopefully this will serve as a good foundation on which to build the rest of my tool!
